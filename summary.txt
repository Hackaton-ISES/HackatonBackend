Tender Guardian backend summary

Core backend
- Created and wired the `tenders` Django app.
- Added Django REST Framework for backend APIs and kept the project backend-only.
- Configured the Unfold admin theme and branded the admin for the tender risk domain.

Domain models
- Implemented `Company`, `Tender`, `TenderRiskAnalysis`, and `RiskReason`.
- Added timestamps, validation, computed properties, and risk score calculation helpers.
- Added migrations for the expanded Tender Guardian schema.

Admin
- Registered all core models in Django admin.
- Added useful list displays, search, filters, ordering, and readonly fields for admin workflows.

API/supporting backend already present
- Tender list, detail, and create endpoints already exist in the project.
- Supporting serializers, tests, and mock seed data were updated to stay compatible with the richer schema.
- Mock data can be seeded with `python manage.py seed_mock_data`.

Repository hygiene
- Improved `.gitignore` to exclude caches, virtualenvs, coverage output, IDE files, build artifacts, and local DB files.
- Removed tracked cache files and `db.sqlite3` from Git tracking so they stop appearing on GitHub after commit/push.

Environment and database
- Switched Django settings from SQLite to PostgreSQL configuration via environment variables.
- Added `.env.example` with the required Django and PostgreSQL variables.
- Added `psycopg` to `requirements.txt` as the PostgreSQL driver dependency.

Containerization
- Added a `Dockerfile` for the Django app image.
- Added `docker-compose.yml` for the Django + PostgreSQL stack.
- Added `.dockerignore` to keep Docker build context clean.
- Added `docker-entrypoint.sh` to wait for PostgreSQL, run migrations, and optionally seed mock data.

Authentication and API expansion
- Added token-based authentication endpoints: `/auth/login`, `/auth/logout`, and `/auth/me`.
- Added `UserProfile` with `admin` and `company` roles linked to Django users and companies.
- Added `/users` registration and admin company listing support.
- Added tender update and delete support while keeping existing tender list/detail/create APIs.
- Added `/applications` endpoints for bid submission, listing, and admin status updates.
- Added `/risk/analyze/:id`, `/risk/stats`, and `/risk/flags/:tenderId` endpoints.

Risk engine improvements
- Extended the scoring engine to cover price comparison, exact budget match, budget overruns, company failure history, winner repetition, consecutive wins, single bidder, short deadlines, late bids, and fake competition patterns.
- Added optional Gemini-generated AI summaries when `GEMINI_API_KEY` is configured.
- Updated serializers and detail responses to include bids, risk analysis, reasons, and AI summary data.

Testing and seed data
- Expanded mock seed data with demo admin/company users and richer tender scenarios.
- Added test coverage for scoring modules, auth flow, applications, risk endpoints, and tender admin actions.
